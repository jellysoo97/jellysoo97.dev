개발을 하다보면 기술의 원리와 필요성을 알지 못한 채 실용적인 사용법만 파고드는 경우가 있다.
보통 트러블슈팅 과정에서 이러한 공부법의 한계가 들어났다.
'이렇게 하니까 그냥 되던데?'스러운 얼렁뚱땅 트러블슈팅을 하다보니 이해했다는 확신이 들지 않았다.
웹 프론트엔드 개발의 전반적인 과정을 돌아보면서 몰랐던 혹은 어렴풋이 알고있던 부분들을 캐치하고 채워나갈 필요성을 느꼈다.
프로그래밍 언어부터 라이브러리까지, 자주 쓰던 기술들의 동작 원리를 체크해보고자 한다.

## 웹 프론트 개발 정리

우선 웹 프론트엔드 개발에 필요한 지식을 4가지로 나눈 다면 다음과 같다고 생각한다.

- 기초: HTML, CSS, JS, (TS)
- 라이브러리, 프레임워크: React, (Vue), (Next.js)
- 데이터 통신(api 호출): REST(axios, tanstack-query, swr), GraphQL
- 상태관리: Context API, Zustand, Redux
- +그 외: 테스트(Storybook, Jest, Vitest, MSW), 버전 관리 시스템(Git) 등등

자주 사용하는 기술 위주로 미흡한 부분을 채워나가고자 한다.

### HTML

> 1. 문서의 구조

모든 웹페이지에는 필수 구성 요소들이 있다. 누락되어도 렌더링하는데 문제가 없긴 하지만, 항상 포함된다.
웹페이지는 크게 `<head>`와 `<body>`로 나눌 수 있다.

- `<head>`의 구성요소
  - 문자 인코딩
  - 문서 제목
  - 메타데이터
  - `<link>`: CSS, favicon, manifest
- `<body>`의 구성요소
  - `<script>`: 자바스크립트 모듈, 브라우저 렌더링 과정과 스크립트 태그 위치의 연관성, `defer`와 `async` 속성
  - HTML 태그

> 2. 시맨틱 HTML

- 시맨틱 태그란
- 코드 블록 콘텐츠의 의미를 나타낼 수 있는가, 의미가 없는 `<div>`, `<span>` 태그를 남용했나
- SEO

> 3. 다양한 태그와 태그별 속성

- 태그: 텍스트(제목, 인용문, 단락), 목록, 링크, 테이블, form, 이미지, 오디오 및 동영상
- 속성: 태그별 속성, 커스텀 속성(`data-`)

- margin, padding

- 스타일 상속

- layout
- paint

- display, flex, grid
- width, max-width, min-width, height, max-height, min-height

- position(relative, absolute, fixed, sticky), z-index, border, overflow, transform, transition, white-space

- CSS-in-JS: styled-components, emotion
- CSS Modules
